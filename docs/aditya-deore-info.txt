When to load:
- Only load and render when the Resume page/tab is active (do not render while hidden).
- Re-render on window resize (debounced) so pages stay fitted to the container width.

Rendering approach:
   140|1. Fetch the PDF document once and cache the loaded document in memory.
2. For each page in the PDF (1 through numPages):
   - Get the page object from PDF.js.
   - Calculate a scale so the page fits the preview container width (max ~860px on desktop, full width on mobile).
   - Create a canvas element for that page.
   - Render the page into the canvas using PDF.js page.render() with a high-DPI output scale (devicePixelRatio, capped at 2–2.5 on mobile for sharp text).
   - Set canvas CSS width to the viewport width and height to auto so it scales responsively.
   - Wrap each canvas in a page container with a label: "Page X of Y".
3. Stack all page containers vertically in a scrollable viewer area so multi-page resumes scroll naturally.

   150|Responsive behavior:
- Phone (container width under ~580px): use full width with no horizontal padding; allow native pinch-to-zoom on the canvas; show hint "Pinch to zoom · Scroll for pages".
- Tablet (under ~768px): slight horizontal padding.
- Desktop: more padding, capped max content width around 860px.

States:
- Loading: show a skeleton/placeholder while the first render is in progress; show "Rendering…" in the preview header.
- Success: show page count in header (e.g. "2 pages"); display all rendered canvases.
- Error: show error message and a direct link/button to download the PDF file.

   160|Performance:
- Debounce initial render slightly (~80ms) after tab becomes active.
- Debounce resize re-renders (~350ms).
- Reuse the cached PDF document on re-render; only redraw canvases when size changes.

Accessibility:

- Page labels should indicate current page and total pages.
